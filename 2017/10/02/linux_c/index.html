如何在linux环境下开始写C语言程序

Linux C语言编程基础

多个源文件共同编译1

hello.c：
#include <stdio.h>
int main()
{
        int a1 = 99;
        int a2 = 100;
        printf("the max num is %d\n",max(a1,a2));
        return 0;
}

max.c：
int max(int a,int b)
{
        if(a>b){
                return a;
        }else{
                return b;
        }
}

作为两个独立的源文件编译时,使用

gcc hello.c max.c -o hello.out

输出可执行文件hello.out,直接./hello.out即可执行。

多个源文件共同编译2

hello.c：
#include <stdio.h>
#include "max.c"
int main()
{
        int a1 = 99;
        int a2 = 100;
        printf("the max num is %d\n",max(a1,a2));
        return 0;
}

max.c：
int max(int a,int b)
{
        if(a>b){
                return a;
        }else{
                return b;
        }
}

在某一个源文件中include另一个源文件时,编译时,使用

gcc hello.c -o hello.out

输出可执行文件hello.out,直接./hello.out即可执行

多个源文件共同编译3

hello.c：
#include <stdio.h>
int main()
{
        int a1 = 99;
        int a2 = 100;
        printf("the max num is %d\n",max(a1,a2));
        return 0;
}

max.c：
int max(int a,int b)
{
        if(a>b){
                return a;
        }else{
                return b;
        }
}

可以先将某些模型编译成obj文件,不产生执行文件

gcc -c max.c -o max.o

然后将max.o和hello.c链接起来使用

gcc max.o hello.c -o hello.out

这样可以加快整个工程的编译速度,因为某些未修改的部分使用的已经编译好的obj文件
我们可以将不会再修改的函数,公共框架,公共类编译生成静态库,将需要修改的业务逻辑部分做成可执行代码。
为了让代码可读性更强,我们可以在.h头文件中建立对相关函数的声明,再其他模块中引入相关头文件。

多个源文件共同编译4(Makefile)

多源程序共同编译,使用gcc xxx.o xxx.o …命令会增加书写复杂度,可以利用Makefile来解决此问题

#this is make file
hello.out:max.o min.o hello.c  
        gcc max.o min.o hello.c -o hello.out
max.o:max.c
        gcc -c max.c
min.o:min.c
        gcc -c min.c

使用#来写注释,hello.out需要文件max.o,min.o,hello.c通过 gcc max.o min.o hello.c -o hello.out命令得到
max.o需要通过max.c文件通过gcc -c max.c得到,依此类推。
书写完之后,执行make命令,会根据Makefile文件来执行里面的相关命令。
之后如果更改了hello.c文件,再此执行make命令时,由于已经编译出max.o和min.o,会省略此步骤。
只执行gcc max.o min.o hello.c -o hello.out操作,可以大大加快编译速度,也比直接使用gcc写一长串来的简单。

main函数中的return

可以使用&&符号来将命令连接起来使用,例如

gcc main.c -o main.out && ./main.out

可以在正确执行完gcc main.c -o main.out操作之后执行&&后面的操作
正确执行就是当前面的命令返回值是0的时候,也就是return 0的时候。
我们可以使用echo $?来查看前一个命令的返回值。
例如：

main.c:
#include <stdio.h>
int main(int argv,char* argc[])
{
        printf("hello\n",);
        return 100;
}
gcc main.c -o main2.out && ./main2.out 
stdout:hello

echo $?
stdout:100

./main2.out && ls
stdout:hello

./main2.out && ls并没有能够正确执行ls命令,因为前一个命令的返回值是100,我们可以利用返回值来进行错误控制

main函数中的参数

#include <stdio.h>

int main(int argv,char* argc[])
{
        printf("argv is :%d\n",argv);
        int i;
        for(i=0;i<argv;i++){
                printf("argc[%d] is %s\n",i,argc[i]);
        }
        return 100;
}

[root@localhost les3]# gcc main3.c -o m3.out
[root@localhost les3]# ./m3.out  1 2 sd 8u
argv is :5
argc[0] is ./m3.out
argc[1] is 1
argc[2] is 2
argc[3] is sd
argc[4] is 8u
[root@localhost les3]#

可以看到我们可以在执行文件后面加入额外参数,这些参数的个数通过int argv传入,具体参数值会以字符串格式按照数组传入char* argc[]里面

标准输入流、输出流以及错误流

stdio.h会提供一系列指向资源的指针。 linux会把一切设备当做文件来处理,当包含的应用程序启动时,
会自动创建三个文件stdin,stdout,stderr,分别用来表示输入流,输出流,错误流。
默认的stdin指向键盘,stdout默认指向显示屏。

实际上printf和scanf是对另外函数的封装

printf("hello\n"); 
fprintf(stdout,"hello\n"); //这二者作用相同

scanf("%d",&a);
fscanf(stdin,"%d",&a); //这二者作用相同

用stderr做错误控制
if(***){
    fprintf(stderr,"error mesaage\n");
    return 1;//not zero
}

标准输入流、输出流以及错误流的重定向

main.c:
#include <stdio.h>
int main()
{
        printf("input the int value i:\n"); 
        int i,j;
        scanf("%d",&i);  
        printf("input the int value j:\n"); 
        scanf("%d",&j); 
        if(0!=j){
                printf("%d/%d=%d\n",i,j,i/j); 
        }else{
                fprintf(stderr,"j!=0\n"); 
                return 1;
        }
        return 0;
}

默认执行方式：
[root@localhost les5]# ./a.out 
input the int value i:
8
input the int value j:
4
8/4=2

输出流重定向：
[root@localhost les5]# ./a.out 1>>a.txt
8
4
[root@localhost les5]# cat a.txt 
input the int value i:
input the int value j:
8/4=2

此方式会把默认输出到显示器上的信息输出到重定向的文件a.txt中,1表示标准输出流,2表示标准错误流,>>表示不覆盖输出,>表示覆盖输出。使用<表示重定向标准输入流,<符号前面不需要数字信息。(不在赘述)例如,我们可以使用:
ls /etc >>etc.txt
把要显示到显示器上的文件目录信息存放在etc.txt文件中。

管道操作

[root@localhost imooc_c++]# ls /etc/ | grep abr
abrt

我们可以使用 | 操作符来实现管道操作,所谓管道就是对前面的输出结果用后面的条件加以过滤,
上述命令的意思就是对ls显示的/etc/目录下的信息,选择包含abr字符串的目录信息输出。

input.c:
#include <stdio.h>

int main()
{
        int flag=1;
        int i;
        int count = 0;
        int s=0;
        while(flag){
                scanf("%d",&i);
                if(0==i) break;
                count++;
                s+=i;
        }
        printf("%d,%d\n",s,count);
        return 0;
}

avg.c:
#include <stdio.h>
int main()
{
        int s,n;
        scanf("%d,%d",&s,&n);
        float v = s/n;
        printf("v= %f\n",v);
        return 0;
}
cc input.c -o input.out 
cc avg.c -o avg.out
[root@localhost les6]# ./input.out | ./avg.out   //可以将具有独立功能的程序利用管道结合起来使用
1000
4000
5000
0
v= 3333.000000

小结

    • 多个源文件编译时候,可以使用include引入.c文件,相当于直接引入代码;
    • 可以利用gcc一起编译执行,每次都需要重新编译多个文件,增加额外开销;
    • 可以先将部分文件编译成obj文件,编译可执行文件时引入其他相关obj文件即可,命令比较复杂繁琐;
    • 编写Makefile文件,执行make命令。并且make命令会忽略掉已经执行完成的部分。编译速度快并且使用简单。
    • 可以使用&&符号来一次执行多个命令,main函数的返回值0用来表示程序被正确执行。
    • main函数的参数(int argv,char* argc[])用来存储执行命令时的参数个数和各个参数的字符串值。
    • 定义了三个流,标准输入流,输出流和错误流。
    • 我们可以对流进行重定向改变输入输出的方式和途径。
    • 我们可以通过管道操作实现各个独立模块的连接过滤。
