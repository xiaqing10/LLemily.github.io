刑侦足迹采集设备开发过程中遇到的问题和解决办法

Introduction

该系统使用cis采集管采集，然后经过A/D芯片将模拟信号转化成数字信号，然后通过两片sdram实现乒乓缓存，最后通过usb芯片传输给PC段进行图像处理。在使用上，通过手工推动设备，在足迹上推动。进行图像采集。

Question1

如何判断开始采集，如何判断采集结束?

Solution1

在设备两侧安装有带有码盘（每转动一周，输出500个脉冲）的滚轮。在上电之后，对码盘脉冲计数，达到一定数量之后（说明设备在进行滚动）开启图像采集。
判断一定时间内的码盘数量低于某个值（说明设备停止运动）时，关闭图像采集。

Question2

因为设备是手持推动，而CIS管的图像采集是逐行扫描的，如果以一个固定时间进行采集，图像会按照推动的快慢在不同的区域出现扭曲变形。

Solution2

  • 我们初期决定通过让CIS按照码盘的速度来进行图像采集，即根据对上一个码盘周期的时间长度来决定CIS管采集的行周期。但是由于实际采集的过程中码盘差距太大，图像质量也不理想。

  • 其次，我们尝试通过把码盘信号附在每行图像信号的左右，即如果当前行持续期间内左边码盘有脉冲，给行起始数据附一个标志，如果右边码盘有脉冲，给行结束数据附一个标志，然后让PC机对图像进行矩阵变换。并且我们还给左右滚轮在机械结构上连接一根支撑杆，保证两边轮子滚动轨迹平行。最后的PC段校正效果也一般，某些连接部分出现锯齿状。图像变得不平整。

  • 最后，我们决定采用机械结构带动CIS管匀速运动的方式，来消除手持推动采集带来的误差，在采集设备悬空覆盖在足迹上，机械结构带动CIS管在被采集对象上运动。可以得到平滑清晰的图像。

证照仪图像采集设备开发过程中遇到的问题和解决办法

Introduction

该系统使用cis采集管采集，然后经过A/D芯片将模拟信号转化成数字信号，然后通过两片sdram实现乒乓缓存，最后通过usb芯片传输给PC段进行图像处理。在使用上，通过插入证照，遮住红外对管，机械结构带动CIS管运动，进行图像采集。机械结构的设计上一定程度的借鉴了刑侦足迹采集设备的经验。

Question1

在图像采集的过程中出现紫外图像很暗。

Solution1

设备上，机械结构带动CIS运动，CIS管扫描证照的过程中，有一个往返的过程，我们在CIS返回的过程中，更改A/D芯片的配置，配置可编程增益放大器的系数。提升紫外的图像的亮度。

xx打印扫描一体机开发过程中遇到的问题和解决办法

Question1

方案的确定:xx公司希望能够实验室提供参数可变的，多dpi工作模式（200+300+600）的图像采集部分的功能，并且他们只能给我们提供数量有限的接口进行图像传输，希望得到的数据流就是正确的图像数据，并且对出图速度（标准是采集结束后）和图像大小做了相关限制。因为最后设备用于银行，对方不希望我们在其电脑上按照任何桌面处理程序。所以必须由实验室在图像处理并压缩，并且支持在线升级等等。

Solution1

  • 第一轮讨论，我们尝试使用实验室老一套方案，DSP+CIS+A/D+SDRAM+USB的方案来提供图像。因为USB的速度够快，能够满足图像速度的要求，通过dsp与打印机交互来对参数做解析，通过emif通信来给FPGA发参数控制图像采集，通过USB来将图像直接传给PC端。这个方案的不足就是，首先，使用USB与PC进行通信，需要在PC端安装驱动，银行出于安全方向的考虑，不希望采用此方案，并且这样传输的图像数据量比较大，一张图像可能达到几十M，不利于存储，xx方希望我们在FPGA实现图像的压缩算法，第一，实现难度较大，并没有相关方面的基础。第二，可能要增加额外的存储空间，增加成本。

  • 第二轮讨论，我们尝试把USB芯片挂在DSP上，用DSP实现图像压缩和传输，DSP实现图像压缩相对FPGA而言简单，也不需要增加硬件成本。实验室DSP组的成员最后讨论说DSP挂USB速度不够，也不知道怎么实现免驱动。

  • 第三轮讨论，我们撤掉了USB芯片，因为免驱的数据传输问题搞不定。决定让打印机给PC传图，由于打印机预留接口有限，我们决定采用spi通信的方式，只需要四个IO就可以实现双向的数据传输。最后定下spi+FPGA+dsp的工作方式。

Introduction

  1. 打印主控板把采集相关的参数通过spi通信方式发送至FPGA
  2. FPGA通过emif通信方式将采集参数发送至DSP解析
  3. DSP解析完成后通过emif通信发送给FPGA，控制FPGA采图
  4. FPGA控制采集图像是个多线程并发过程，包括:
    • FPGA图像采集控制
    • FPGA将图像数据通过视频口发送给DSP
    • DSP对图像进行压缩，并通过emif通信发送给FPGA
    • FPGA将收集到的数据通过spi通信方式转发给打印机控制板

Questions

  1. spi通信方式的调试。
    Spi通信的测试是此方案的关键，采集指令和参数以及最后的图像数据都通过这个方式传输，需要在保证一定速度的同时，保证100%的准确率。

  2. 中断的控制
    中断的控制，由于在我们使用的dsp芯片上，只有两个中断源，实际在使用的过程中，当FPGA接收完打印机发送的指令和参数后，需要通过中断的方式通知dsp，FPGA的数据是逐行采集，图像通过视频口的方式发送给DSP,每行图像结束后有个行中断。以及DSP最后发送图像数据给FPGA，FPGA存储完成通知SPI取走数据后，再次通过中断的方式通知DSP继续发送下一个包的数据。如何让只有两个中断源的DSP处理多个中断事件。

  3. 图像预处理
    由于实际CIS管的数据流是三段数据的，即先数据第一个像素，然后输出 行像素点/3+1，行像素点*2/3+1，所以数据流并不是一副标准图像，需要对数据流进行处理，保证输出的数据顺序和图像对应。如果用DSP直接操作内存的话，整个过程十分简单，但是由于DSP读取整副图像内存耗时校正。最后决定用FPGA来处理图像。

  4. 600dpi模式的实现
    由于xx公司需要dpi参数可配置的图像采集模式，我们需要做200dpi，以及600dpi的测试，200dpi模式相对简单，直接更改CIS管的配置，以及其他参数即可。我们在对一行图像做乒乓处理的过程中，需要缓存一行，并且乒乓操作，需要开辟可以存储两行的内存空间，而内部ram的内存空间是以2的幂次方来开辟的，600dpi图像，一行有5184个点，上下管10368个点，只能用16K存储，乒乓一下即需要32K的内部ram资源，而我们使用的芯片标明只有33.75K，如果采用16K来做600dpi图像处理，工作都无法正常编译。而增加内部ram需要更换FPGA芯片，增加大量成本。

  5. 图像的校正
    在图像采集过程中，由于CIS管的各个传感器的感光特性不完全一致，实际上一副白纸采集出现来的像素值可以介于200-230之间，而对RGB各个分量采集如果都存在些许误差的话，最后合成的彩色图像可能会有偏色，虽然偏色现象不明显，但是银行方也不能荣誉，我们考虑对采集到颜色做一个校正处理，即用标准的白色卡纸进行采集，然后采集像素和最后实际像素做一个y=kx+b的线性变化，依此来抵消CIS管传感器采集的细微偏差，此方案之前也用于PC端的处理，能够把偏色很严重的图像校正回来，与实际物品相差无几。而对一行进行逐点校正，需要存储一行的K参数和B参数。而昨晚图像处理之后已经内有足够的ram资源来存储校正参数。

  6. 多dpi工作后内部ram的精简
    要让一套系统支持多个dpi工作模式，需要在编译的过程同时考虑不同dpi的图像处理过程，我认为需要对不同的dpi模式开辟单独的ram空间来对图像进行处理，那么之前讨论的33.75实际上只能用来处理单一dpi模式，不符合xx方的设计要求。而且增加内部ram的方案价格上也不被接受。

Solutions:

  1. 通过与打印机主控板的交流，规定好发送数据的起始和结束标识，数据总长度可变，保证发送参数的灵活和方便，将参数存进FPGA芯片的内部ram中供dsp读出进行解析。通过和dsp的交流，规定好一次只发送一个512字节的数据包，将数据存进内部ram中供spi主机读取。调试spi通信状态机的切换，在dsp对FPGA进行读写操作时，由dsp提供读写时钟操作ram，由dsp提供读写地址，在保持地址不变的条件下，对读写的异步时钟与系统时钟进行与操作，对同一个地址多次读写，以免异步时钟扰动出现漏读漏写的情况。

  2. 我们分析发现，spi取走数据包和cis管图像采集行中断是同时发生的，因为最终的系统需要做到，一边采集，一边处理，一边校正，一边压缩，一边发送。所有操作是同时进行的，所以我们把这两个中断事件放在两个中断源，而FPGA接收完参数包是提前发生的，我们将此中断和取数据中断集成在一起，通过控制emif的某标志地址上的数据值来指明当前中断的中断类型。此方面扩展了中断源能够响应的事件个数，对于不同时发生的中断，可以很方便的扩展。

  3. CIS管的数据是逐行输出的，我们也逐行来对数据流做校正，而且内部ram的速度完全可以操作CIS的数据流输出，不会增加额外的处理时间。我们采用乒乓缓存的方式，将图像数据流调整之后存进一片ram中，然后操控第二行去写另外一片ram，与此同时，将第一片ram中的数据写入到视频口去。这个操作使整个系统（300dpi工作模式下）的速度提升了4-5秒。

  4. 最后，我们决定采用8+4的模式，即将600dpi模式的图像一行分开存储，用一个8K+4K的内存进行存储，这样乒乓之后所用的内存空间只需要24K。能够正常运行。

  5. 考虑到更换FPGA芯片正在芯片内部ram的成本比较贵，我们决定在FPGA外部外挂一个SRAM，此SRAM的读取速度可以达到10ns,地址线宽17bit（128KB）,数据线宽8bit,我们CIS管输出的数据流是48M*8M，对一个点同时取出K和B参数，需要达到96M的读速度，10ns可以满足需求。我们对各个dpi模式的RGB各个分量的校正系数进行存储（这里的存储地址可以连续存放，由于sram的地址线只有一套，不能边读边写，不适合拿来作图像预处理）。在采集过程中，根据当前采集的灰度类型，取出对应的校正参数，经过校正之后的图像色彩更符合实物。

  6. 实际上在使用过程中，一个时刻只会采集一种dpi模式的图像，和之前复用中断源一样，我们考虑对内部ram进行复用，根据dpi参数，用选择器选择进入ram的数据，控制ram的地址，数据的存储方式等等，ram的读取方式以及进入校正处理的选择，最后成功的将多种dpi的数据处理方式复用在一起。在不增额外内存资源开销的同时，扩展了功能。
