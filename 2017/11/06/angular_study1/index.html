《angular.js权威指南》学习笔记1

一些认识

angular是一种前端的MVC框架，将数据(视图)和表现分离开来，减少了重复的DOM操作，主要关注对数据的处理。扩展了HTML的功能。angular的特点，双向绑定和依赖注入。

简单指令

>

  1. ng-app=” “ 定义angularJS的使用范围；
  2. ng-init=”变量=值;变量=’值’” 初始化变量的值，有多个变量时，中间用分号隔开；
  3. ng-model=”变量” 定义变量名；
  4. ng-bind=”变量” 绑定变量名，获取该变量的数据。这里的变量就是第3条的变量名。但是一般都用双重花括号来获取变量的值，比如：{ {变量}}。

    其中ng-init和ng-bind都是不建议使用的，ng-init初始化会使得结构臃肿，并且把数据和元素混在一起，不符合表现和数据分离的特点。
    ng-bind不能绑定表达式，扩展性不好，使用{ { }}符号来提取ng-model更使用。

开始使用angular

<!DOCTYPE html>
<html>
<head>
<meta charset="utf-8">
<script src="http://cdn.static.runoob.com/libs/angular.js/1.4.6/angular.min.js"></script>
</head>
<body>
<div ng-app="">
     <p>名字 : <input type="text" ng-model="name"></p>
     <h1>Hello {{name}}</h1>
</div>
</body>
</html>

需要通过script标签引入angular框架，通过ng-app定义应用的范围，在页面比较复杂的情况下，可以使用控制器来分离

<div ng-app="myApp" ng-controller="myCtrl">

名: <input type="text" ng-model="firstName"><br>
姓: <input type="text" ng-model="lastName"><br>
<br>
姓名: {{firstName + " " + lastName}}

</div>

<script>
var app = angular.module('myApp', []);
app.controller('myCtrl', function($scope) {  
    /*默认必须有一个$scope对象，控制器内的各个视图模型以及函数全部被定义在这个$scope作用域上，如果需要引入其他服务，增加其他对象即可，例如function($scope,$http) */
    $scope.firstName= "John";
    $scope.lastName= "Doe";
});
</script>

{ { }}表达式和其他常见指令

>
AngularJS 表达式写在双大括号内：{ { expression }}。
AngularJS 表达式把数据绑定到 HTML。
AngularJS 将在表达式书写的位置”输出”数据。
AngularJS 表达式 很像 JavaScript 表达式：它们可以包含文字、运算符和变量。
实例 { { 5 + 5 }} 或 { { firstName + “ “ + lastName }}

ng-repeat 指令用来重复一个HTML元素

<div ng-app="" ng-init="names=['Jani','Hege','Kai']">
  <p>使用 ng-repeat 来循环数组</p>
  <ul>
    <li ng-repeat="x in names">
         {{ x }}
    </li>
  </ul>
</div>

ng-repeat指令用在一个对象数组上

<div ng-app="" ng-init="names=[
{name:'Jani',country:'Norway'},
{name:'Hege',country:'Sweden'},
{name:'Kai',country:'Denmark'}]">

<p>循环对象：</p>
<ul>
  <li ng-repeat="x    in names">
    {{ x.name + ', ' + x.country }}
  </li>
</ul>

</div>

ng-show或者ng-hide用来定义显示或者隐藏某个元素

通过给ng-show或者ng-hide赋布尔类型的值来定义显示隐藏
html布尔属性只要出现即默认true,例如disabled,readonly,checked,selected等等，前面加上ng-之后，变量angular类型的属性，可以通过布尔值来定义该属性的有效或者无效

<input type="text" ng-model='an'><br/>
<input type="text" ng-disabled='!an'><br/>
<hr>  //当在第一个输入框输入数据时，第二个框才能被使用

<input type="text" ng-model="pro"><br/>
<input type="text" value="some text" ng-readonly="pro"><br/>  
<hr>  //当在第一个框输入数据时，第二个框变成只读模式

$scope作用域

Scope(作用域) 是应用在 HTML (视图) 和 JavaScript (控制器)之间的纽带。
Scope 是一个对象，有可用的方法和属性。
Scope 可应用在视图和控制器上。
当你在 AngularJS 创建控制器时，你可以将 $scope 对象当作一个参数传递:

<div ng-app="myApp" ng-controller="myCtrl">
<h1>{{carname}}</h1>
</div>

<script>
var app = angular.module('myApp', []);
app.controller('myCtrl', function($scope) {
    $scope.carname = "Volvo";
});
</script>

在大型项目中,HTML DOM 中有多个作用域,所有的应用都有一个 $rootScope，它可以作用在 ng-app 指令包含的所有 HTML 元素中。
$rootScope 可作用于整个应用中。是各个 controller 中 scope 的桥梁。用 rootscope 定义的值，可以在各个 controller 中使用。$rootscope设置的变量在所有controller里面都是可以直接用{ {$root.变量名}}来显示的，当然也可以赋值给$scope.

过滤器

过滤器可以使用一个管道字符（|）添加到表达式和指令中。

//将数组转换成大写
<div ng-app="myApp" ng-controller="personCtrl">

<p>姓名为 {{ lastName | uppercase 
}}

</div>

除了内置的过滤器之外，可以自定义过滤器

//将msg逆序输出
<!DOCTYPE html>
<html>
<meta charset="utf-8">
<script src="http://cdn.static.runoob.com/libs/angular.js/1.4.6/angular.min.js"></script>

<body>

<div ng-app="myApp" ng-controller="myCtrl">


姓名: {{ msg | reverse }}

</div>

<script>
var app = angular.module('myApp', []);
app.controller('myCtrl', function($scope) {
    $scope.msg = "Runoob";
});
app.filter('reverse', function() { //可以注入依赖
    return function(text) {
        return text.split("").reverse().join("");
    }
});
</script>

</body>
</html>

AngularJS 服务

AngularJS 中你可以创建自己的服务，或使用内建服务。

//例如使用$location服务返回当前页面的地址
var app = angular.module('myApp', []);
app.controller('customersCtrl', function($scope, $location) {
    $scope.myUrl = $location.absUrl();
});

$http 是 AngularJS 应用中最常用的服务。 服务向服务器发送请求，应用响应服务器传送过来的数据。

var app = angular.module('myApp', []);
app.controller('myCtrl', function($scope, $http) {
    $http.get("welcome.htm").then(function (response) {
        $scope.myWelcome = response.data;
    });
});

也可以自定义服务

app.service('hexafy', function() {
    this.myFunc = function (x) {
        return x.toString(16);
    }
});  //自定义一个用于转换成16进制数的服务
app.controller('myCtrl', function($scope, hexafy) {
    $scope.hex = hexafy.myFunc(255);
});

http服务

$http 是 AngularJS 中的一个核心服务，用于读取远程服务器的数据。封装了xhr请求
在学习这一块的过程中，有一个认识就是，如果我们用表单提交的get和post请求，用表单的action属性去响应事件，适用于页面需要跳转的情况，如果仍停留在当前页面，则返回值会覆盖原页面。如果想通过请求对页面做局部更新，则需要用到ajax方法，因为定义了XMLHttpRequest 对象相当于给出了一个用于接收返回数据的对象，可以通过对返回值的处理，来更新页面。而如果通过表单来处理的话，则没有用于处理响应数据的接口，会导致整个页面被返回数据重新覆盖。所以是否动态刷新页面关键在于前端对响应所做的处理。与后端的处理没有关系。而在angularjs的http请求中，已经封装了ajax方法。
在后续的使用中，除非要进行页面跳转的情况下，否则少直接用表单提交来进行通信。

存在一个疑问：使用form表单向本页面做出请求(内置有PHP代码)的时候，是否也进行了页面的重绘？还是只对echo部分做了局部更新

使用格式：
// 简单的 GET 请求，可以改为 POST
$http({
    method: 'GET',
    url: '/someUrl'
}).then(function successCallback(response) {
        // 请求成功执行代码
    }, function errorCallback(response) {
        // 请求失败执行代码
});


var app = angular.module('myApp', []);

app.controller('siteCtrl', function($scope, $http) {
    $http({
        method: 'GET',
        url: 'https://www.runoob.com/try/angularjs/data/sites.php'
    }).then(function successCallback(response) {
            $scope.names = response.data.sites;
        }, function errorCallback(response) {
            // 请求失败执行代码
    });

});
//使用promise类型？来对响应做出处理

可以在get请求中加入params属性，请求会在url后以键值对的形式发送

$http({
    method: 'GET',
    url: 'https://www.baidu.com',
    params:{action:'data'}
})
在f12调试栏看到的请求就是'https://www.baidu.com?action=data  
在服务器端需要通过req.query.action来获取

可以在get请求的url路径中进行追加

$http({
    method: 'GET',
    url: 'https://www.baidu.com/lol'
})
在f12调试栏看到的请求就是'https://www.baidu.com/lol 
在服务器端可以用app.get('/:bala',....)来对此路径做出响应(假设根目录对www.baidu.com)
我们可以通过req.params.bala获取到值lol
